# tencent-abrplayer

`tencent-abrplayer` contains four parts:

- `backend/`: unified Go HTTP service (Gin + GORM + MySQL). Serves the
  abrplayer static assets, the admin dashboard, health checks, the
  bandwidth probe endpoint, telemetry ingestion (`/api/stat/*`),
  admin-only stats query APIs, and the WebRTC play URL helper. Built
  to run as `abrplayer-backend` via Docker.
- `multipusher/`: Fyne desktop app that publishes five SRT streams to Tencent Cloud Live.
- `abrplayer/`: JS/WebRTC player static assets and ABR logic.
- `admin-web/`: management console served by `backend` at `/dashboard/`.

## Stream ladder

For each `siteName`, `multipusher` publishes five streams:

| Level | Stream name | Codec | Target bitrate | Selection rule |
| --- | --- | --- | --- | --- |
| bottom | `{siteName}_audio` | AAC audio only | 128k | `bitrate < 300 kbps` |
| economic | `{siteName}_economic` | H.264 | 400k | `bitrate < 600 kbps` |
| standard_hevc | `{siteName}_standard_hevc` | HEVC | 600k | standard HEVC preferred when supported |
| standard | `{siteName}_standard` | H.264 | 1000k | `bitrate < 1500 kbps` |
| high | `{siteName}` | H.264 | 2000k | `bitrate >= 1500 kbps` |

The highest quality stream name is exactly `{siteName}`. Do not append `_high`.
Tencent SRT publishing uses AAC 128k audio by default because Tencent Cloud playback does not preserve Opus audio in this SRT live path. Startup logs include
`[FFMPEG][AudioDebug]` and `[FFMPEG][AudioProbe]` entries to verify selected
audio input, stream mapping, and output codec.

## Multipusher

The Fyne UI is the only runtime entry point. It supports:

- `siteName` selection with linked stream names.
- device input mode: select enumerated video and audio devices.
- file input mode: select an `.mp4` file for local testing.
- five-stream output view: stream name, encoding parameters, and per-stream publish status.
- `Start`, `Stop`, and `Play` actions.

Configuration is stored in YAML at:

```text
multipusher/conf/pusher.local.yml
```

A template lives at `multipusher/conf/pusher.{env}.yml`. The actual `pusher.{local,test,uat,stag,prod}.yml` files are gitignored because they contain Tencent push secrets. Override via env vars `VPUBLISHER_APP_ID`, `VPUBLISHER_APP_SECRET`, `VPUBLISHER_TX_KEY_MAIN`, `VPUBLISHER_TX_KEY_BACK`.

Tencent Cloud SRT URLs are generated by the program:

```text
srt://{host}:{port}?streamid=#!::h={host},r={app}/{streamName},txSecret={txSecret},txTime={txTime}
```

`txSecret` and `txTime` are generated by `multipusher/utils/tencent.go`.

## Backend service

The backend listens on port `8088` by default (configurable via the
`HttpPort` field in `conf/backend.{env}.json`).

Open the player at:

```text
http://<server>:8088/
```

The root path redirects to `/abrplayer/index.html`. The admin console
lives at `/dashboard/` and requires a session login created via
`POST /api/auth/login`.

Public endpoints (no auth, rate-limited):

```text
GET  /healthz
GET  /api/stat/health
GET  /bandwidth-probe
GET  /api/settings/statapi-domain
POST /api/auth/login
POST /api/auth/logout
GET  /api/auth/me
POST /api/stat/play/start
POST /api/stat/play/end
POST /api/stat/lag
POST /api/stat/record
POST /api/stat/playback
```

Auth-protected (session cookie or Bearer token):

```text
GET  /api/play/txUrl
GET  /api/settings/play-domain
PUT  /api/settings/play-domain
GET  /api/settings/env-domains
PUT  /api/settings/env-domains
POST /api/stat/round/{start,end,cancel}
POST /api/stats/play/{list,trend}
GET  /list/{fields,fields/streams,hourly,daily,fieldround,round}
```

Environment public base URLs are stored in the MySQL `info_system_settings`
table and editable from the admin dashboard. Code-side defaults for the
five supported `APP_ENV` values:

| APP_ENV | Label | Public base URL |
| --- | --- | --- |
| `local` | local-env | `http://localhost:8088` |
| `dev` | UAT-env | `https://videostat-uat.example.com` |
| `test` | test-env | `https://videostat-test.example.com` |
| `stag` | STAG-env | `https://videostat-stag.example.com` |
| `prod` | PROD-env | `https://videostat-prod.example.com` |

Legacy `uat` runtime input is accepted and normalized to `dev`. AbrPlayer
calls `GET /api/settings/statapi-domain` against its own backend to
discover the active environment URL, then builds telemetry URLs like
`{domain}/api/stat/play/start`.

## Bandwidth probe

ABR bandwidth probing is based only on downloading static files. It does not
use codec information, and it does not treat HEVC/H.264 differently.

Current probe target:

1. Same-origin `NIOES8001.jpeg` under the `abrplayer` static root.

The browser reads up to `120KB` or for up to `3s`, then estimates:

```text
kbps = downloaded_bytes * 8 / elapsed_ms
```

The backend also exposes `GET /bandwidth-probe?bytes=N` (default 1 MB,
maximum 5 MB) that streams synthetic bytes for clients that need a fixed
server-controlled probe payload.

If all probe assets are unavailable, the result is `null`. In that case
the player keeps the current stream and does not downgrade to audio.
Codec compatibility is handled separately by player errors, including
HEVC fallback to H.264.

## Docker deployment

The root `Dockerfile` packages `backend/` plus the `abrplayer` and
`admin-web` static assets. It does not include the Fyne `multipusher`
desktop UI. The runtime base image is `jrottenberg/ffmpeg:6.1-alpine`
because the backend's stream-status cron uses `ffprobe`.

Build:

```powershell
docker build -t abrplayer-backend:latest .
```

Run locally:

```powershell
docker run --rm -p 8088:8088 abrplayer-backend:latest
```

The image supports the following environment variables:

```text
ABRPLAYER_ROOT=/opt/abrplayer        # static asset root
STATAPI_SESSION_SECRET=<32-byte hex> # session signing key (strongly recommended)
STATAPI_ADMIN_USERNAME=<user>        # overrides admin.username in config
STATAPI_ADMIN_PASSWORD=<pass>        # overrides admin.password in config
STATAPI_TX_KEY_MAIN=<tencent main key>
STATAPI_TX_KEY_BACK=<tencent backup key>
```

MySQL connection settings live in `conf/backend.{env}.json` under the
`Mysql` block. Apply the schema with:

```bash
mysql -h <host> -u root -p -e "CREATE DATABASE IF NOT EXISTS <db_name> CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;"
mysql -h <host> -u root -p <db_name> < backend/models/init_schema.sql
```

The DB name is read from `Mysql.DbName` at runtime; the schema script is
DB-name agnostic.

## Build and test

From `backend`:

```powershell
go test ./apis/... ./models/... ./tracer/...
```

From `multipusher`:

```powershell
go test ./utils/... ./conf/... ./tracer/... ./ws/...
```

Windows Fyne builds require CGO and a MinGW compiler. The project build
script is:

```powershell
sh build.sh
```

On Windows the MSYS2 entry point used for build checks is:

```powershell
& 'D:\prog\msys64\mingw64.exe' -defterm -no-start -here -c 'cd /d/projs/tencent-abrplayer/multipusher && sh build.sh'
```
