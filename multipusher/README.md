# multipusher

`multipusher` is a Fyne desktop app that captures one local video/audio source and pushes five fixed SRT live streams to Tencent Cloud Live for ABR playback.

The Fyne UI is the only runtime entry point. It replaces the old CLI flow.

## Stream ladder

For `siteName=3drush-fwh`, the app publishes:

| Level | Stream name | Codec | Target bitrate |
| --- | --- | --- | --- |
| bottom | `3drush-fwh_audio` | AAC audio only | 128k |
| economic | `3drush-fwh_economic` | H.264, 360x640 portrait | 400k |
| standard_hevc | `3drush-fwh_standard_hevc` | HEVC | 600k |
| standard | `3drush-fwh_standard` | H.264 | 1000k |
| high | `3drush-fwh` | H.264 | 2000k |

The high stream name is exactly `{siteName}`.

The five stream parameters are YAML-configurable under `streams`. If `streams`
is omitted, the app uses the defaults shown above.

## Tencent SRT URL

The app builds Tencent Cloud SRT URLs automatically:

```text
srt://{host}:{port}?streamid=#!::h={host},r={app}/{streamName},txSecret={txSecret},txTime={txTime}
```

`txSecret` and `txTime` are generated by `utils/tencent.go`. Each stream name gets its own signature.

Example:

```text
srt://publish.example.com:9000?streamid=#!::h=publish.example.com,r=live/3drush-fwh,txSecret=...,txTime=...
```

## YAML config

Default config path:

```text
conf/pusher.local.yml
```

A template lives at `conf/pusher.{env}.yml`. Real configs are gitignored because they hold Tencent push secrets.

Example:

```yaml
siteName: 3drush-fwh

tencentSrt:
  host: publish.example.com
  port: 9000
  app: live
  tokenDays: 30

input:
  mode: device
  videoDevice: OBS Virtual Camera
  audioDevice: Microphone Array (Realtek(R) Audio)
  videoFile: ""
  videoLayout: portrait

streams:
  - level: bottom
    streamName: "{siteName}_audio"
    audioOnly: true
    audioCodec: aac
    audioBitrate: 128k
  - level: economic
    streamName: "{siteName}_economic"
    audioOnly: false
    videoCodec: h264
    videoBitrate: 400k
    videoMaxrate: 600k
    portraitWidth: 360
    portraitHeight: 640
    landscapeWidth: 640
    landscapeHeight: 360
    audioCodec: aac
    audioBitrate: 128k
  - level: standard
    streamName: "{siteName}_standard"
    audioOnly: false
    videoCodec: h264
    videoBitrate: 1000k
    videoMaxrate: 1500k
    portraitWidth: 720
    portraitHeight: 1280
    landscapeWidth: 1280
    landscapeHeight: 720
    audioCodec: aac
    audioBitrate: 128k
  - level: high
    streamName: "{siteName}"
    audioOnly: false
    videoCodec: h264
    videoBitrate: 2000k
    videoMaxrate: 3000k
    portraitWidth: 1080
    portraitHeight: 1920
    landscapeWidth: 1920
    landscapeHeight: 1080
    audioCodec: aac
    audioBitrate: 128k

publish:
  onReady: false
  reconnectMinSeconds: 3
  reconnectMaxSeconds: 60
```

Tencent SRT publishing uses AAC 128k audio because Tencent Cloud playback does not preserve Opus audio in this SRT live path. Startup logs include
`[FFMPEG][AudioDebug]` and `[FFMPEG][AudioProbe]` lines to confirm the selected
audio input, mandatory audio map, and output codec.

`publish.onReady` controls whether the UI starts publishing as soon as the app
is running. The default is `false`, so startup only loads the UI unless this is
explicitly set to `true`.

The embedded ABR player server is independent of publisher startup. If automatic
publishing fails, `multipusher` keeps running so it can still serve as the local
ABR player engine.

## Run

Requirements:

- Go 1.24.1+
- `ffmpeg` available in `PATH`
- Windows DirectShow devices for camera/audio capture, or an `.mp4` file for file input mode

Start the UI:

```powershell
go run .
```

Build:

```powershell
sh build.sh
```

Use the UI to select the site, choose device or file input mode, review the four output streams, and start/stop publishing.

Current UI actions:

- `Start`: validate and save YAML, then start all four publishers.
- `Stop`: stop all publishers.
- `Play`: open the local ABR player URL.
- `Publish on UI start`: save `publish.onReady`; when true, the next UI startup starts publishing automatically.

The Four-level output section shows each stream's name, encoding parameters, and publish status.

AbrPlayer backend is a separate service under `../backend`. It listens on:

```text
0.0.0.0:8088
```

Remote users should open `http://<server-ip-or-domain>:8088/?site=<siteName>`.

The backend must be reachable through the host firewall or cloud security group on TCP `8088`.

The Multipusher `Play` button opens `http://127.0.0.1:8088/` with the current
stream query parameters. Start the backend separately before using `Play`.

The backend persists per-environment public base URLs in MySQL and exposes:

```text
GET /api/settings/env-domains
PUT /api/settings/env-domains
GET /api/settings/statapi-domain
```

MySQL connection settings live in `backend/conf/backend.{env}.json` under the
`Mysql` block; see `backend/README.md` for schema setup details.

For local testing, switch `input.mode` to `file` and select an `.mp4` file in the UI:

```yaml
input:
  mode: file
  videoFile: D:\videos\test.mp4
  videoLayout: portrait
```
